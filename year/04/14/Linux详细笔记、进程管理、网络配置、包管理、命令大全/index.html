一、Linux目录结构

  • bin：存放二进制命令文件，该文件夹本身不是一个文件夹，而是usr/bin文件夹的链接。
  • sbin：存放系统级的二进制命令文件，s(system)，同样是一个文件夹的链接，usr/sbin。
  • lib：存放库文件，链接到usr/lib。
  • lib64：存放64位相关的库文件，链接到usr/lib64。
  • usr：存放用户所有的应用程序、文件和数据。
  • boot：挂载引导分区的目录，存放引导启动的文件。
  • dev：设备目录，存放管理系统设备的文件。
  • etc：存放系统管理所需要的配置文件。
  • home：存放除root用户外其他用户的主目录。均以用户名命名。
  • root：root用户的主目录。
  • opt：存放第三方软件包的文件夹，安装软件的文件夹。
  • media：识别可移动媒体设备，可移动媒体设备的挂载点。
  • mnt：移动设备的另一个挂载点。
  • proc：进程目录，存放进程信息。
  • run：运行目录，存放系统运行以来的所有实时信息。
  • srv：存放跟系统服务相关的数据。
  • sys：存放系统硬件相关的数据。
  • tmp：零时文件存放目录。
  • var：可变目录，存放不断变化的文件，如日志文件。

三、网络配置

VMware的三种网络连接模式

  1. 桥接模式
    虚拟机直接链接外部物理网络的模式，主机起到了网桥的作用。这种模式下，虚拟机可以直接访问外部网络，并且对外部网络是可见的。
    1. NAT模式
      虚拟机和主机构建一个专用网络，并通过虚拟网络地址转换（NAT）设备对IP进行转换。虚拟机通过共享主机IP可以访问外部网络，但外部网络无法访问虚拟机。
    2. 仅主机模式
      虚拟机只与主机共享一个专用网络，与外部网络无法通信。

Linux 系统中为了提高磁盘的读写效率，对磁盘采取了 “预读迟写”操作方式。当用户保存文件时，Linux 核心并不一定立即将保存数据写入物理磁盘中，而是将数据保存在缓冲区中，待缓冲区满时再写入磁盘，这种方式可以极大的提高磁盘写入数据的效率。但是，也带来了安全隐患，如果数据还未写入磁盘时，系统断电或出现其他严重问题，则将导致数据丢失。使用sync指令可以立即将缓冲区的数据写入磁盘。

五、文件权限类

  1. 文件权限

Linux系统是一种典型的多用户系统，不同的用户处于不同的地位，拥有不同的权限。为了保护系统的安全性，Linux系统对不同的用户访问同一文件或目录的权限做了不同的规定。在Linux中我们可以使用ll或者ls -l命令来显示一个文件的属性以及文件所属的用户和组。

从左到右的 10 个字符表示

  • 第1位表示文件类型，-代表文件、d代表目录、l代表链接
  • 第1-3位表示文件所有者拥有该文件的权限
  • 第4-6位确定属组（所有者的同组用户）拥有该文件的权限
  • 第7-9位确定其他用户拥有该文件的权限

  1. 修改文件或目录权限：chmod

    方式一：chmod [{ugoa}{+-=}{rwx}] 文件或目录
    方式二：chmod [mode=421 ] [文件或目录]

    u：所有者 g：所属组 o：其他人 a：所有人(u、g、o 的总和)

  2. 修改文件或目录的所有者：chown [参数] [最终用户] [文件或目录] ，参数可选-R递归操作

  3. 修改文件或目录的所属组：chgrp [最终用户组] [文件或目录]

每个*代表的含义

含义 范围
第一个 “ * ” 一小时的第几分钟 0-59
第二个 “ * ” 一天的第几小时 0-23
第三个 “ * ” 一个月的第几天 1-31
第四个 “ * ” 一年的第几月 1-12
第五个 “ * ” 一周的星期几 0-7，0和7都表示7

特殊符号

符号 含义
* 表示任何时间，如第一个“*”代表一小时中每分钟都执行一次
, 表示不连续的时间，如“0 8,12,16 * ”，代表每天的8点0分，12 点0分，16点 0分都执行一次
- 表示连续的时间范围，如“0 5 1-6 ”，代表周一到周六的凌晨 5 点 0 分执行
*/n 表示每隔多久执行一次，如“/10 * ”，代表每隔 10 分钟执行一次

七、软件包管理

一、RPM

RPM（RedHat Package Manager），RedHat软件包管理工具。

  1. RPM 查询命令（rpm -qa）：查询安装的所有 rpm 软件包
  2. RPM 卸载命令（rpm -e）：卸载软件包
  3. RPM 安装命令（rpm -ivh RPM包全名）
参数 功能
-i install，安装
-v —verbose，显示详细信息
-h —hash，进度条

二、YUM

YUM（全称为 Yellow dog Updater, Modified）是一个在 Fedora 和 RedHat 以及 CentOS 中的 Shell 前端软件包管理器。基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包 并且安装，可以自动处理依赖性关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装

语法：yum [选项] [参数]

选项 功能
-y 对所有安装过程中需要确认的位置自动确认
参数 功能
install 安装 rpm 软件包
update 更新 rpm 软件包
remove 删除指定的 rpm 软件包
check-update 检查是否有可用的更新 rpm
list 显示软件包信息
clean 清理 yum 过期的缓存
deplist 显示 yum 软件包的所有依赖关系
